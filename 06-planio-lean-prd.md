# Planio 5-Step Lean PRD

> A concise process inspired by Marty Cagan's original PRD structure. Focuses on what is being built, not how.
> "A PRD spells out your destination. It is not a map of how to get there."
> Best for: Agile teams wanting structure without heavyweight docs, bridging high-level requirements with sprint-level work.

## Step 3: Release Criteria

_How will you know when this is ready for release?_

| Criterion | Minimum Acceptable Standard |
|-----------|-----------------------------|
| **Functionality** | What must be included? Essential features that can't be missing? |
| **Usability** | How easy/effective do core functions need to be? |
| **Reliability** | How consistent should performance be? What failure rate is acceptable? |
| **Performance** | Minimum expected speed/responsiveness? |
| **Supportability** | What ongoing maintenance is realistic? |
